Runtime functions in configuration files
========================================

The ALSA library can modify the configuration at runtime. Several built-in functions are available.

A function is defined with the id ``@func`` and the function name. All other values in the current compound are used as configuration for the function. If the compound func.<function_name> is defined in the root node, then the library and function from this compound configuration are used, otherwise 'snd_func\_' is prefixed to the string and code from the ALSA library is used. The definition of a function looks like:

.. ref-code-block:: cpp

	func.remove_first_char {
	    lib "/usr/lib/libasoundextend.so"
	    func "extend_remove_first_char"
	}



.. _doxid-conffunc_1conffunc_ref:

Function reference
~~~~~~~~~~~~~~~~~~

* The getenv function - :ref:`snd_func_getenv() <doxid-confmisc_8c_1a033eab97ee6619aeb8c5eca585acf6c0>` - obtains an environment value. The result is a string.

* The igetenv function - :ref:`snd_func_igetenv() <doxid-confmisc_8c_1a121864fba02478b9f605be44c97f5dc8>` - obtains an environment value. The result is an integer.

* The concat function - :ref:`snd_func_concat() <doxid-confmisc_8c_1a43a3685d5e397fdc0e46af6d7b121616>` - merges all specified strings. The result is a string.

* The iadd function - :ref:`snd_func_iadd() <doxid-confmisc_8c_1a4e7acbc261ca8b6bf4602753ea5c17fe>` - sum all specified integers. The result is an integer.

* The imul function - :ref:`snd_func_imul() <doxid-confmisc_8c_1a12f4b8b67e0976a260859edb929f4d36>` - multiply all specified integers. The result is an integer.

* The datadir function - :ref:`snd_func_datadir() <doxid-confmisc_8c_1aa24c85e86b4c1f6c4d2d440e4782d098>` - returns the ALSA data directory. The result is a string.

* The refer function - :ref:`snd_func_refer() <doxid-confmisc_8c_1a8d91af8387a708c526bac3ee2c7ea10e>` - copies the referred configuration. The result has the same type as the referred node.

* The card_inum function - :ref:`snd_func_card_inum() <doxid-confmisc_8c_1a023f9fd40df7c2c428a457b6ec7f8004>` - returns a card number (integers).

* The card_driver function - :ref:`snd_func_card_driver() <doxid-confmisc_8c_1a2d147e387821449a8927d60ef4f8ff16>` - returns a driver identification. The result is a string.

* The card_id function - :ref:`snd_func_card_id() <doxid-confmisc_8c_1a9ccfc361f1bc1d81837335de02707ee8>` - returns a card identification. The result is a string.

* The card_name function - :ref:`snd_func_card_name() <doxid-confmisc_8c_1aba1ae0185e2d3e9b50291b40e7048181>` - returns a card's name. The result is a string.

* The pcm_id function - snd_func_pcm_id() - returns a pcm identification. The result is a string.

* The private_string function - :ref:`snd_func_private_string() <doxid-confmisc_8c_1af14b528e2502f2037294301473389948>` - returns the string from the private_data node.

* The private_card_driver function - :ref:`snd_func_private_card_driver() <doxid-confmisc_8c_1a7bc4fec8cb3499997934a00210d34fe7>` - returns the driver identification from the private_data node. The result is a string.

* The private_pcm_subdevice function - snd_func_private_pcm_subdevice() - returns the PCM subdevice number from the private_data node. The result is a string.
