	snd_seq_query_port_subscribers(
		:ref:`snd_seq_t<doxid-group___sequencer_1gae141149958a7458806152ed5942a1acd>`* seq,
		:ref:`snd_seq_query_subscribe_t<doxid-group___seq_subscribe_1ga67932dfbdaa8fc71a08d058fa7121870>`* subs
	)

query port subscriber list

Queries the subscribers accessing to a port. The query information is specified in subs argument.

At least, the client id, the port id, the index number and the query type must be set to perform a proper query. As the query type, :ref:`SND_SEQ_QUERY_SUBS_READ <doxid-group___seq_subscribe_1ggafd3bd807eb95e26760bced605f7b4b9fa0dbe48892b30a0412c19a304d05dd6e6>` or :ref:`SND_SEQ_QUERY_SUBS_WRITE <doxid-group___seq_subscribe_1ggafd3bd807eb95e26760bced605f7b4b9fa62a1002a52629b24aba14f592e8e0689>` can be specified to check whether the readers or the writers to the port. To query the first subscription, set 0 to the index number. To list up all the subscriptions, call this function with the index numbers from 0 until this returns a negative value.



.. rubric:: Parameters:

.. list-table::
	:widths: 20 80

	*
		- seq

		- sequencer handle

	*
		- subs

		- subscription to query



.. rubric:: Returns:

0 on success otherwise a negative error code
